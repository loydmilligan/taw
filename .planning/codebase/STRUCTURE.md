# Codebase Structure

**Analysis Date:** 2026-04-14

## Directory Layout

```
taw/
├── src/                    # Source code (TypeScript)
│   ├── cli/                # Entry points: app shell, bridge, research harness
│   ├── app/                # Ink/React TUI components and state
│   ├── commands/           # Slash command definitions (~40 commands)
│   ├── core/               # Business logic: sessions, chat, providers, modes, etc.
│   ├── services/           # External integrations: config, filesystem, logging, wiki, search
│   ├── types/              # TypeScript interfaces and schemas
│   ├── utils/              # Utilities: ID generation
│   └── version.ts          # App version constant
├── dist/                   # Compiled JavaScript (generated by tsc)
├── tests/                  # Vitest unit and integration tests (~20 test files)
├── docs/                   # Project documentation (architecture, specs, roadmap)
├── infra/                  # Docker compose for SearXNG dev container
├── browser-extension/      # Browser capture extension (scaffolding)
├── tsconfig.json           # TypeScript config (strict mode, ES2022, NodeNext)
├── vitest.config.ts        # Vitest config (node environment)
├── eslint.config.js        # ESLint flat config
├── .prettierrc.json        # Prettier formatting (semi, single-quote, no trailing comma)
├── package.json            # Dependencies and build scripts
├── pnpm-lock.yaml          # Locked dependency versions
└── .gitignore              # Excludes dist/, node_modules/, .env, .ai/
```

## Directory Purposes

**`src/`:**
- Purpose: All source code (TypeScript)
- Contains: CLI, TUI, commands, core business logic, services, types, utils
- Key files: `version.ts` (app version)

**`src/cli/`:**
- Purpose: Application entry points
- Contains: `entry.tsx` (main TUI), `bootstrap.ts` (app init), `bridge.ts` (HTTP server), `research-harness.ts` (headless mode)
- Key files: `entry.tsx` (main), `bootstrap.ts` (config load, session create)

**`src/app/`:**
- Purpose: Ink/React TUI rendering and state management
- Contains: `App.tsx` (main component), layout components, state utilities
- Key files:
  - `App.tsx` - Main render loop, keyboard handling, command dispatch
  - `state.ts` - Input state helpers (insertText, deleteBackward, cursor movement)
  - `theme.ts` - Terminal color/style definitions
  - `layout/Header.tsx` - Status rail (mode, provider, session info)
  - `layout/Transcript.tsx` - Chat history display
  - `layout/Footer.tsx` - Action hints and keyboard shortcuts
  - `layout/CommandPalette.tsx` - Command/mode palette (Ctrl+L)
  - `layout/MapPanel.tsx` - Brainstorm map visualization
  - `layout/MapPicker.tsx` - Load/save maps UI
  - `components/InputBar.tsx` - Text input field

**`src/commands/`:**
- Purpose: Slash command definitions (~40 commands)
- Contains: Individual command files, parser, registry, types
- Key files:
  - `registry.ts` - Master command list + lookup function
  - `parser.ts` - Tokenize user input, extract command/args
  - `types.ts` - CommandDefinition interface
  - Command files: `brainstorm.ts`, `research.ts`, `workflow.ts`, `wiki.ts`, `finalize.ts`, etc.

**`src/core/`:**
- Purpose: Core business logic layers
- Contains: Sessions, chat, providers, modes, prompts, artifacts, context, research, telemetry, tools, notes

**`src/core/chat/`:**
- Purpose: Conversation management
- Key files: `engine.ts` - Main chat turn executor (message building, provider call, tool dispatch)

**`src/core/providers/`:**
- Purpose: AI provider abstraction layer
- Key files:
  - `index.ts` - Provider selector function
  - `openai-compatible-provider.ts` - OpenAI and compatible APIs
  - `anthropic-provider.ts` - Anthropic API

**`src/core/sessions/`:**
- Purpose: Session management
- Key files:
  - `session-manager.ts` - Create, load, persist sessions
  - `schema.ts` - Zod schemas for session metadata

**`src/core/modes/`:**
- Purpose: Mode definitions and system prompts
- Key files:
  - `definitions.ts` - Mode registry (name, artifact type, activation message)
  - `src/core/prompts/modes.ts` - Mode-specific system prompts

**`src/core/artifacts/`:**
- Purpose: Artifact creation and tracking
- Key files: `writer.ts` - Write markdown artifacts, update session metadata

**`src/core/context/`:**
- Purpose: Prompt context assembly
- Key files:
  - `prompt-context.ts` - Build full context string (agents, soul, memory, docs)
  - `assistant-files.ts` - Load global/project agent rules and summaries

**`src/core/research/`:**
- Purpose: Web search and source management
- Key files:
  - `search.ts` - SearXNG client, query and store results
  - `store.ts` - Persist sources to JSON, rating system
  - `source-rating.ts` - User source quality ratings
  - `schema.ts` - Zod schemas for research data

**`src/core/tools/`:**
- Purpose: LLM-callable local tools (wiki write, search, etc.)
- Key files: `runtime.ts` - Tool invocation dispatcher

**`src/core/telemetry/`:**
- Purpose: Usage tracking and cost calculation
- Key files:
  - `collector.ts` - Record per-request metrics
  - `store.ts` - Persist telemetry to session JSON
  - `derivation.ts` - Aggregate session usage

**`src/core/notes/`:**
- Purpose: Session notes persistence
- Key files: `notes-writer.ts` - Append conversation to notes.md

**`src/services/`:**
- Purpose: External service integrations
- Contains: Config, filesystem, logging, wiki, search, hister, openrouter, testing

**`src/services/config/`:**
- Purpose: Configuration loading and validation
- Key files:
  - `loader.ts` - Load global + project config with precedence
  - `schema.ts` - Zod schemas for GlobalConfig, ProjectConfig
  - `env.ts` - Environment variable reading

**`src/services/filesystem/`:**
- Purpose: Centralized file I/O
- Key files: Wrappers for fs/promises operations

**`src/services/wiki/`:**
- Purpose: Local markdown wiki management
- Key files:
  - `manager.ts` - Topic discovery, file R/W, mode parsing
  - `reindex.ts` - Rebuild wiki index from files
  - `link-review.ts` - Validate and fix cross-links
  - `hister-ingest.ts` - Browser history ingest from Hister
  - `frontmatter.ts` - YAML front matter parsing

**`src/services/search/`:**
- Purpose: Search backend management
- Key files: `searxng-manager.ts` - Docker container control, health checks

**`src/services/logging/`:**
- Purpose: Debug and error logging
- Key files: `logger.ts` - Daily log files, API key redaction

**`src/bridge/`:**
- Purpose: HTTP bridge for browser extension and headless automation
- Key files:
  - `server.ts` - Route handlers (research ingest, wiki ops, job dispatch)
  - `launcher.ts` - Launch background sessions
  - `payload-file.ts` - Write results to files

**`src/types/`:**
- Purpose: TypeScript interfaces
- Key files:
  - `app.ts` - AppState, TranscriptEntry, BrainstormMap
  - `provider.ts` - ProviderAdapter interface, ChatMessage, ProviderConfig
  - `session.ts` - SessionRecord, SessionMetadata (if exists)

**`src/utils/`:**
- Purpose: Utility functions
- Key files: `ids.ts` - Snowflake-like ID generation with prefix

**`dist/`:**
- Purpose: Compiled JavaScript output
- Contains: Mirrored `src/` structure as .js files
- Generated by: `pnpm build` (TypeScript compiler)
- Committed: No (in .gitignore)

**`tests/`:**
- Purpose: Unit and integration tests
- Contains: ~20 test files in Vitest format
- Key files:
  - `session-manager.test.ts`
  - `config-loader.test.ts`
  - `command-parser.test.ts`
  - `provider-smoke.test.ts`
  - `wiki-*.test.ts` (wiki system tests)
  - `bridge-server.test.ts`

**`docs/`:**
- Purpose: Project documentation
- Contains: Architecture, UI spec, command hierarchy, testing strategy, roadmap
- Key files:
  - `architecture.md` - System design overview
  - `ui-ux-spec.md` - Terminal design principles
  - `tasks.md` / `tasklist.md` - Implementation milestones
  - `command-hierarchy.md` - Command categorization
  - `testing-strategy.md` - QA approach

**`infra/`:**

**`browser-extension/`:**
- Purpose: Browser capture tooling (scaffolding)
- Contains: HTML, CSS, JS for Chrome extension

## Key File Locations

**Entry Points:**
- `src/cli/entry.tsx`: Main TUI app entry
- `src/cli/bridge.ts`: HTTP bridge entry
- `src/cli/research-harness.ts`: Headless automation entry

**Configuration:**
- `.ai/config.json`: Project-scoped config (not committed, user creates)
- `~/.config/taw/config.json`: Global user config
- `tsconfig.json`: TypeScript compiler options
- `vitest.config.ts`: Test runner config
- `.prettierrc.json`: Code formatter options
- `eslint.config.js`: Linter rules

**Core Logic:**
- `src/core/chat/engine.ts`: Chat turn execution
- `src/core/sessions/session-manager.ts`: Session lifecycle
- `src/core/providers/`: AI provider implementations
- `src/commands/`: All user-callable commands
- `src/app/App.tsx`: Main TUI render loop

**Testing:**
- `tests/`: All test files
- `vitest.config.ts`: Test config

## Naming Conventions

**Files:**
- kebab-case for module files: `session-manager.ts`, `command-parser.ts`, `chat-engine.ts` (not used; `engine.ts` is preferred)
- PascalCase for React component files: `App.tsx`, `Header.tsx`, `InputBar.tsx`
- Lowercase for service files: `logger.ts`, `logger.ts` in service folders
- Command files named by command: `brainstorm.ts`, `research.ts`, `workflow.ts`

**Directories:**
- lowercase with hyphens: `src/core/chat/`, `src/services/filesystem/`, `src/commands/`
- No underscores in directory names

**Functions/Constants:**
- camelCase: `executeChatTurn()`, `createSession()`, `buildPromptContext()`
- PascalCase for React components: `<App />`, `<Header />`, `<Transcript />`
- SCREAMING_SNAKE_CASE for constants: `WIKI_WRITE_MIN_TOKENS = 4096`

**Exports:**
- Named exports preferred
- Default exports rare (only for React components in app/layout/)
- Barrel files: `src/commands/index.ts`, `src/core/providers/index.ts`

## Where to Add New Code

**New Feature (chat-related):**
- Primary code: `src/commands/` (if slash command) or `src/core/` (if logic layer)
- Chat integration: Update `src/core/chat/engine.ts` if new tool or prompt needed
- Tests: `tests/` with name matching feature

**New Command:**
- Implementation: `src/commands/{command-name}.ts`
- Export from: `src/commands/registry.ts` (add to commandRegistry array)
- Type: Use `CommandDefinition` interface from `src/commands/types.ts`
- Pattern: Export named handler function, default export CommandDefinition

**New Mode:**
- Definition: Add entry to `src/core/modes/definitions.ts` (export getModeDefinition)
- System prompt: Add case to `src/core/prompts/modes.ts` (buildModeSystemPrompt)
- Artifact type: Set in mode definition (artifactType, artifactTitle)

**New Component:**
- Location: `src/app/layout/` (if layout-related) or `src/app/components/` (if reusable)
- Pattern: React functional component, export as named or default
- Props: Interface at top of file (e.g., `interface HeaderProps { ... }`)

**New Service:**
- Location: `src/services/{service-name}/` (new subdirectory)
- Main file: `{service-name}.ts` or `manager.ts`
- Exports: Functions that hide implementation details
- Pattern: No singletons; pass config/session as arguments

**Utilities:**
- Shared helpers: `src/utils/`
- Service-specific helpers: Keep in service folder

**Tests:**
- Location: `tests/` (mirror src structure loosely)
- Naming: `{feature-name}.test.ts`
- Pattern: Vitest describe/it, use mocks from `src/services/testing/`

## Special Directories

**`.ai/`:**
- Purpose: Project session storage and config (user creates on init)
- Contains: `config.json`, `sessions/`, `notes.md`, artifacts
- Generated: Yes (by init command)
- Committed: No (.gitignore)
- User-modifiable: No (except config.json)

**`~/.config/taw/`:**
- Purpose: Global user config, sessions (non-project mode), logs, cache
- Contains: `config.json`, `sessions/`, `logs/`, optional cache
- Generated: Yes (by app on first run)
- Committed: No (user-local)
- User-modifiable: Yes (config.json)

**`node_modules/`:**
- Purpose: Installed npm dependencies
- Generated: Yes (by pnpm install)
- Committed: No (.gitignore)

**`dist/`:**
- Purpose: Compiled JavaScript (output of tsc)
- Generated: Yes (by pnpm build)
- Committed: No (.gitignore)
